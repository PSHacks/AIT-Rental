# Rental-Fleet System

Современное веб-приложение для аренды автомобилей. Проект включает в себя клиентский интерфейс для выбора и бронирования авто, а также защищенную панель администратора для управления парком и заказами.

## Стек технологий

### Языки и Фреймворки
- **TypeScript**: Полная типизация Frontend и Backend.
- **Backend**: Node.js + Express.
- **Frontend**: React.js + Vite.

### База данных и ORM
- **PostgreSQL (Neon)**: Облачная реляционная СУБД.
- **Drizzle ORM**: Управление схемой и выполнение типобезопасных запросов.
- **Zod**: Валидация данных на всех этапах.
- **TanStack Query (React Query)**: Эффективное управление серверным состоянием.

---

## Архитектура базы данных

1. **Таблица cars**: Хранит информацию о транспортных средствах (модель, категория, цена в сутки, серийный номер, статус доступности).
2. **Таблица bookings**: Данные о бронировании (связь с автомобилем, даты аренды, статус заказа).
3. **Таблица reviews**: Отзывы и рейтинги пользователей для каждой единицы флота.

---

## Функциональные возможности

- **Для клиентов:**
  - Просмотр списка доступных автомобилей с фильтрацией (Economy, Comfort, Luxury).
  - Проверка доступности авто на конкретные даты в реальном времени.
  - Создание бронирования.
  - Система отзывов и рейтингов.
- **Для администраторов:**
  - Авторизация.
  - Удаление (soft delete) автомобилей.
  - Просмотр всех бронирований и обновление их статусов (Pending/Confirmed/Cancelled).

## API Endpoints (Порт: 5051)

### Публичные методы
- `GET /api/cars` — Получение списка всех доступных машин.
- `POST /api/bookings/check` — Проверка доступности (JSON: carId, startDate, endDate).
- `POST /api/bookings` — Создание новой брони.
- `GET /api/reviews/:carId` — Просмотр отзывов.

### Защищенные методы (требуют заголовок Authorization)
- `DELETE /api/cars/:id` — Удаление автомобиля из системы.
- `GET /api/admin/bookings` — Полный список заказов.
- `PATCH /api/admin/bookings/:id` — Изменение статуса бронирования.

## Установка и запуск

1. **Клонируйте репозиторий.**
2. **Установите зависимости**: `npm install`
3. **Настройте файл .env**:
   Добавьте ваш `DATABASE_URL` (Neon) и `PORT=5051`.
4. **Синхронизируйте БД**: `npm run db:push`
5. **Запустите проект**: `npm run dev`
